# Руководство по вкладу в проект

Мы рады вашему интересу к участию в развитии нашего CLI проекта. Проект открыт для вклада от сообщества, и мы ценим ваше участие. Просьба внимательно ознакомиться с данным руководством, чтобы ваша работа была интегрирована максимально эффективно.

## Процесс участия

1. **Форк репозитория**

   Создайте форк репозитория нашего проекта на GitHub, чтобы иметь в своем доступе полную копию актуального кода.

2. **Клонирование репозитория**

   Локально клонируйте ваш форк:

   bash
   git clone https://github.com/HSE-SD-2025/CLI.git

3. **Создание Feature Branch**

   Перед началом работы создайте отдельную ветку для ваших изменений. Используйте осмысленное именование для лучшей идентификации задач:

   bash
   git checkout -b feature/название-задачи

## Справочник стиля и тестирование

- Весь код должен соответствовать нашему [Руководству по стилю](STYLE_GUIDE.md). Это гарантирует согласованность и читаемость.
- Новые функции должны сопровождаться соответствующими тестами. Все существующие тесты должны успешно проходить до и после ваших изменений.

## Порядок внесения изменений

1. **Разработка и локальная проверка**

   Реализуйте изменения на локальной машине и убедитесь, что все тесты проходят успешно:

   ```bash
   ./gradlew test
   ```

2. **Подготовка изменений**

   Добавьте и закоммитьте свои изменения. Цель каждого коммита — быть независимым и понятным:

   ```bash
   git add .
   git commit -m "Описание изменений с указанием причины и деталей реализации"
   ```

3. **Обработка пулл-реквеста**

   Запушьте коммиты в ваш fork и создайте пулл-реквест. В описании PR должно быть четко указано, что вы изменили и почему.

4. **Согласование изменений**

   Все пулл-реквесты требуют обязательного согласования с одним из ведущих разработчиков: [ilia-anikeev](https://github.com/ilia-anikeev), [andrewbolotsky](https://github.com/andrewbolotsky) или [MaksimkaSH](https://github.com/MaksimkaSH). Пожалуйста, пометьте их для ревью вашего PR.

## Обратная связь и обсуждения

- Мы открыты для предложений и изменений. Чтобы избегать дублирования или ненужной работы, прежде чем приступать к значительным изменениям, создайте запрос на обсуждение через [Issues](https://github.com/HSE-SD-2025/CLI/issues).

## Лицензирование

Участие в проекте означает согласие с текущими условиями лицензирования. Убедитесь, что вы прочитали и поняли [лицензионное соглашение](LICENSE).
