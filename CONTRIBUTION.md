## Как внести вклад

### Отчёт об ошибках и предложения по улучшениям
   Найдена ошибка? Заведите новый issue с чётким описанием шага воспроизведения и ожидаемым поведением.
   Есть предложение или новый функционал? Откройте issue с пометкой "feature request" и кратко опишите идею.
### Как сделать Pull Request
1. Форкните репозиторий и создайте отдельную ветку под вашу задачу:
   ```bash
   git checkout -b my-feature-branch
   ```
2. Сделайте изменения с понятными и чистыми коммитами.
3. Протестируйте свой код:

   Убедитесь, что все автоматические тесты проходят
   
      Проверьте, что изменения не ломают существующую функциональность.
   
      Оформите код согласно стилю проекта.
   
      Возвращайтесь к основной ветке и обновите её перед отправкой PR:
   
      ```bash
      git checkout main
      git pull origin main
      git checkout my-feature-branch
      git rebase main
   ```
   Отправьте Pull Request на GitHub и подробно опишите изменения.
4. Code Review
   Ваш Pull Request будет рассмотрен одним из мейнтейнеров.
   Возможно, вам предложат правки или зададут уточняющие вопросы.
   Старайтесь реагировать на комментарии максимально быстро.
5. Приём изменений
   После одобрения PR будет слит в основную ветку.
   Вы будете указаны в благодарностях проекта!
6. Стиль и рекомендации
   Держите изменения небольшими по объёму — "одна задача/фича — один PR".
   Описывайте смысл коммитов кратко и по существу.
   Пиши комментарии, если часть кода требует пояснений.
   Спасибо за вклад!
   Будем рады любой помощи, независимо от вашего уровня.
   Если есть вопросы, пишите в issues или обратитесь к одному из мейнтейнеров.
