# Руководство по стилю Java

Настоящий документ описывает стандарты и рекомендации по стилю кодирования на языке Java для нашего проекта. Эти правила помогут поддерживать согласованность и читаемость кода.

## Общие правила

- **Форматирование кода:** Используйте стандартное [форматирование Java](https://google.github.io/styleguide/javaguide.html). Настройте ваше IDE для автоматического применения автоформатирования перед каждым коммитом.
- **Отступы:** Используйте 4 пробела для отступов. Не используйте табуляции.
- **Длина строки:** Максимальная длина строки — 100 символов. Строки длиннее следует разбивать.
- **Кодировка:** Все исходные файлы должны быть в UTF-8.

## Именование

- **Классы и интерфейсы:** Используйте PascalCase. Пример: `MyClass`, `EmployeeData`.
- **Методы:** Используйте camelCase. Пример: `getEmployeeName`, `calculateSalary`.
- **Переменные и поля:** Используйте camelCase. Пример: `employeeId`, `totalSalary`.
- **Константы:** Используйте заглавные буквы с подчеркиванием. Пример: `MAXIMUM_VALUE`, `DEFAULT_TIMEOUT`.

## Организация кода

- **Один класс на файл:** Каждый класс должен находиться в своём собственном файле.
- **Порядок расположения содержимого класса:**
    1. Константы
    2. Поля
    3. Конструкторы
    4. Методы публичные
    5. Методы приватные и защищенные

## Комментарии

- **Классы и методы должны иметь Javadoc комментарии.** Используйте комментарии к методам для объяснения их функциональности и параметров.
- **Внутри метода:** Комментируйте сложные или неочевидные фрагменты кода с использованием комментариев типа `//`.

## Практики кодирования

- **Исключения:** Используйте исключения для обработки ошибок. Всегда указывайте значимые сообщения об ошибках.
- **Работа с коллекциями:** Следуйте правилам безопасного использования коллекций, предпочтительно интерфейсы (например, `List`, `Map`) вместо конкретных классов (например, `ArrayList`, `HashMap`).
- **Null-значения:** Минимизируйте использование `null`. Используйте `Optional` из Java 8 и новее.

## Стиль кодирования

- **Фигурные скобки:** Используйте фигурные скобки для всех блоков, даже если они содержат один оператор.

- **Пробелы:** Обязательно используйте пробел до и после операторов, таких как `=`, `+`, `-` и других бинарных операторов.

## Конфигурация IDE

- **Настрока форматирования:** Убедитесь, что ваше интегрированное средство разработки (IDE) настроено на использование описанных стилей форматирования.
- **Инструменты анализа кода:** Рекомендуется использование инструментов статического анализа кода, таких как Checkstyle или SonarQube, для проверки соответствия кода стилю.

Мы настаиваем на строгом соблюдении данных правил во всех коммитах. Это обеспечит единый стиль в кодовой базе и облегчит совместную работу над проектом.
