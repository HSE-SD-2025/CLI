# CLI Интерпретатор

## Описание проекта

Этот проект представляет собой простой интерпретатор командной строки (CLI), который поддерживает базовые команды, такие как `cat`, `echo`, `wc`, `pwd`, `exit`, `grep`. Дополнительно поддерживает работу с переменными окружения и обеспечивают возможность вызова внешних программ, а также пайплайны для передачи данных между командами.

## Архитектура

Основные компоненты системы:

- **CLI Engine**: Основной компонент, отвечающий за обработку ввода пользователя, разбор строк и выполнение команд.
- **Команды**: Интерфейс команд с реализациями для каждой поддерживаемой команды (`EchoCommand`, `CatCommand`, `WcCommand`, `ExitCommand`, `PwdCommand`, `GrepCommand`, `ExternalCommand`).
- **Переменные окружения**: Средство хранения и управления окружением с поддержкой подстановок.

Диаграмма структуры:

```plaintext
CLI Engine
├── Commands
│   ├── Command Interface
│   │   ├── EchoCommand
│   │   ├── CatCommand
│   │   ├── WcCommand
│   │   ├── ExitCommand
│   │   ├── PwdCommand
│   │   ├── GrepCommand
│   │   └── ExternalCommand
```

### Поддерживаемые команды

- `cat [FILE]`: Выводит содержимое файла на экран.
- `echo [ARGS]`: Выводит аргументы на экран.
- `wc [FILE]`: Выводит количество строк, слов и байт в файле.
- `pwd`: Показывает текущую директорию.
- `exit`: Выходит из интерпретатора.
- `grep [OPTIONS] PATTERN [FILE...]`: Ищет PATTERN в указанных файлах.

#### Опции grep:
- `-w`: Поиск только целых слов
- `-i`: Регистронезависимый поиск
- `-A NUM`: Выводить NUM строк после каждой строки с совпадением

Работа с одинарными и двойными кавычками, переменными окружения и оператором `|` для создания пайплайнов также поддерживается.

## Выбор библиотеки для разбора аргументов

Для реализации команды `grep` был выбран модуль `argparse` из стандартной библиотеки Python по следующим причинам:

1. **Встроенность**: `argparse` является частью стандартной библиотеки Python, что означает отсутствие дополнительных зависимостей.
2. **Богатый функционал**: Поддерживает все необходимые типы аргументов (позиционные, опциональные, флаги), валидацию, справку и т.д.
3. **Простота использования**: Имеет интуитивно понятный API и хорошо документирован.
4. **Гибкость**: Позволяет легко добавлять новые опции и модифицировать существующие.
5. **Стандартизация**: Является де-факто стандартом для разбора аргументов командной строки в Python.
