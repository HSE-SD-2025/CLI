# Архитектурная Документация CLI Интерпретатора

## 1. Введение

### 1.1. Цель
Описание архитектуры простого интерпретатора командной строки (CLI), реализующего базовые команды и функциональности для обработки пользовательского ввода на основе стандарта IEEE 42010:2011.

### 1.2. Область применения
Описывается архитектура системы CLI, охватывая основные функциональные элементы, их взаимодействие и поддерживаемые команды.

## 2. Архитектурное описание

### 2.1. Основные компоненты

- CLI Engine: Центральный компонент, который отвечает за прием ввода от пользователя, его разбор и выполнение команд.
- Команды: Интерфейс и конкретные реализации команд (`EchoCommand`, CatCommand, WcCommand, ExitCommand, PwdCommand, `ExternalCommand`), выполняющиеся через CLI.
- Переменные окружения: Хранилище для управления данными окружения и поддержка подстановок.

### 2.3. Архитектурные Взгляды

#### 2.3.1. Логический Взгляд

CLI Engine парсит введенную пользователем команду, разбивая её на последовательные действия, такие как выполнение отдельных команд и обработка пайплайнов. Каждый компонент командной архитектуры реализует интерфейс, обеспечивающий однородное выполнение.

#### 2.3.2. Процессный Взгляд

Система поддерживает запускаемые команды в виде отдельных процессов, используя пайплайны для маршрутизации вывода одного процесса на вход другого. Это позволяет командам взаимодействовать и передавать данные параллельно.

#### 2.3.3. Физический Взгляд

CLI развертывается как программное обеспечение, исполняющееся на хост-операционной системе, которая поддерживает многозадачность и функции межпроцессного взаимодействия.

### 2.4. Подкомпоненты и их взаимодействия

#### 2.4.1. CLI Engine

- Управляет разбором строк, администрированием выполнения и логикой пайплайнов.
- Поддерживает вызовы внешних программ и управление переменными окружения.

#### 2.4.2. Команды 

- Command Interface: Определяет метод execute(String[]) для всех команд, что обеспечивает единый способ их запуска.
  
- EchoCommand, CatCommand, WcCommand, ExitCommand, PwdCommand: Реализуют базовые команды командной строки.

- ExternalCommand: Позволяет вызывать и управлять внешними программами из внутри CLI.

### 2.5. Переменные Окружения

- Обрабатываются в контексте парсинга и используются для подстановок значений в командах.
- Хранятся в совокупности данных типа Map<String, String>.

### 2.6. Потоки и Пайплайны

- Команды могут быть связаны в последовательности через механизм пайплайнов, передавая данные из одной команды в другую, используя стандартные потоки ввода-вывода.
- Поддержка одинарных и двойных кавычек, и операторов для создания пайплайнов.

## 3. Заключение
Архитектура CLI интерпретатора описана в соответствии с IEEE 42010:2011, обеспечивая полное представление её компонентов, взаимодействий и функциональности.

## 4 Поддерживаемые команды

- cat [FILE]: Выводит содержимое файла на экран.
- echo [ARGS]: Выводит аргументы на экран.
- wc [FILE]: Выводит количество строк, слов и байт в файле.
- pwd: Показывает текущую директорию.
- exit: Завершение работы интерпретатора.
- Поддержка работы с одинарными и двойными кавычками, переменными окружения и оператора | для создания пайплайнов.
