## 1. Общее описание
   Система представляет собой однопользовательский интерпретатор командной строки (CLI engine), реализующий чтение пользовательского ввода, его разбор, обработку переменных окружения, исполнение встроенных и внешних команд (с поддержкой пайплайнов).

Точка входа: метод run() класса CliEngine.

## 2. Структурная диаграмма
   ```plaintext
   +-----------------------------------------------------+
   |                     CliEngine                       |
   +-----------------------------------------------------+
   | - env: dict                                        |
   | - is_exit: bool                                    |
   | - return_code: int                                 |
   +-----------------------------------------------------+
   | + run()                                            |
   | - __parse_commands(command_line: str)              |
   | - __parse_command(command: str)                    |
   | - __execute_piped_commands(commands_list: list)    |
   | - __substitute_env(command: str)                   |
   +-----------------------+-----------------------------+
                           |
                           |
                           v
               +-----------------------------+
               |   commands (package)        |
               +-----------------------------+
               |  commands: dict[str, CI]    |
               +-----------------------------+
                           |
                           v
                 +---------------------------------------------+
                 |               CommandInterface              |
                 +---------------------------------------------+
                 | + execute(args)                             |
                 | + execute_with_input(args, stdin) (необяз.) |
                 +---------------------------------------------+
               /       |            |             |           |  \   \
              /        |            |             |           |   \    \___________________
            /          |            |             |           |     _______                \
           v           v            v             v           v             v               v
   +-----------+ +-----------+ +-----------+ +-----------+ +-----------+ +-----------+ +-----------+
   |EchoCommand| |CatCommand | |ExitCommand| |External...| |GrepCommand| |PwdCommand | |WcCommand  |
   +-----------+ +-----------+ +-----------+ +-----------+ +-----------+ +-----------+ +-----------+
```
## 3. Описание работы системы
   ### 3.1. Парсинг строки ввода
   Ввод пользователя читается в основном цикле методом run().
   Ввод разбивается на сегменты по символу | — это "пайплайновые" команды.
   Для каждого сегмента:
   Сначала производится подстановка переменных окружения ($VAR) через регулярное выражение.
   Затем сегмент токенизируется через shlex.split() (корректная обработка кавычек).
   Если токен выглядит как присваивание переменной (имя=значение), переменная добавляется в env и команда не исполняется.
   Иначе список токенов добавляется для дальнейшего исполнения.
   ### 3.2. Работа с переменными окружения
   Переменные хранятся в self.env: dict.
   Можно создавать, переопределять на лету через присваивание (VAR=значение) в командной строке.
   Подстановки ($VAR) выполняются перед токенизацией для каждого сегмента.
   ### 3.3. Создание и исполнение команд
   Все команды реализуют интерфейс (CommandInterface) с методом execute (и опционально execute_with_input).
   По имени в списке токенов выбирается соответствующая реализация через словарь commands.
   Если команда не является встроенной — используется ExternalCommand (обёртка на системный вызов).
   В пайплайнах аргументам команд дополнительно передаётся стандартный ввод (выход предыдущей команды).
   ### 3.4. Пайплайны
   Исполнение происходит последовательно: вывод предыдущей команды передаётся на ввод следующей через переменную previous_output.
   Реального многопроцессного пайплайна нет — обработка строго последовательная.
   ### 3.5. Код возврата и завершение
   Переменная return_code хранит статус последней выполненной команды.
   ExitCommand меняет is_exit на True, что завершает основной цикл.
## 4. Описание компонентов
   ### 4.1. CommandInterface и команды
   Интерфейс:
      1. execute(args: list[str])
      2. execute_with_input(args: list[str], stdin) (опционально)
   Команды:
      1. EchoCommand
      2. CatCommand
      3. ExitCommand
      4. ExternalCommand
      5. GrepCommand
      6. PwdCommand
      7. WcCommand

 ### 4.2. Хранение результата парсинга
 Результат парсинга — список списков (каждый подсписок — команда с аргументами).
 ### 4.3. Обработка ошибок
     Все ошибки исполнения команд и парсинга выводятся пользователю.
## 5. Поток исполнения
   Точка входа: CliEngine.run()
   Приём ввода → токенизация → обработка присваиваний/команд → создание и исполнение объектов-команд → возврат/вывод результата → завершение при exit.
