

### Архитектурное описание CLI-интерпретатора в формате IEEE 42010:2011

---

## 1. Введение  
**Цель документа**: Описать архитектуру однопользовательского CLI-интерпретатора, поддерживающего выполнение встроенных и внешних команд, пайплайны, управление переменными окружения и обработку ошибок.  
**Область применения**: Разработка, модификация и анализ системы интерпретатора командной строки.  
**Определения**:  
- **Пайплайн** — последовательная передача вывода одной команды на вход другой.  
- **Токенизация** — разбор строки ввода на отдельные элементы (токены).  
- **CommandInterface** — абстрактный интерфейс для реализации команд.  

---

## 2. Контекст системы  
**Роль системы**: CLI-интерпретатор для выполнения команд с поддержкой пайплайнов, переменных окружения и обработки ввода/вывода.  
**Взаимодействие с окружением**:  
- **Пользователь**: Вводит команды через консоль.  
- **ОС**: Исполнение внешних команд через системные вызовы.  
- **Внутренние данные**: Переменные окружения хранятся в памяти интерпретатора.  

**Стейкхолдеры и их интересы**:  
- **Разработчики**: Расширяемость, модульность, простота тестирования.  
- **Пользователи**: Удобство ввода, поддержка пайплайнов, корректная обработка переменных.  
- **Администраторы**: Надежность, предсказуемость кодов возврата.  

---

## 3. Архитектурные цели и ограничения  
**Цели**:  
1. Корректный парсинг и выполнение команд.  
2. Поддержка последовательных пайплайнов.  
3. Динамическое управление переменными окружения.  
4. Расширяемость за счет добавления новых команд.  

**Ограничения**:  
1. Отсутствие параллелизма при выполнении пайплайнов.  
2. Обработка ошибок ограничивается выводом сообщений.  

**Architectural Drivers**:  
- **Функциональные требования**: Поддержка встроенных/внешних команд, пайплайнов, переменных окружения.  
- **Качественные требования**: Простота расширения (через CommandInterface), предсказуемость выполнения.  

---

## 4. Архитектурные представления  

### 4.1. Логическая структура (Logical View)  
**Компоненты**:  
- **`CliEngine`** (основной класс):  
  - **Атрибуты**:  
    - `env: dict` — переменные окружения.  
    - `is_exit: bool` — флаг завершения работы.  
    - `return_code: int` — код возврата последней команды.  
  - **Методы**:  
    - `run()`: Основной цикл обработки ввода.  
    - `__parse_commands()`: Разбиение строки на команды по пайплайнам.  
    - `__substitute_env()`: Подстановка переменных окружения.  
    - `__execute_piped_commands()`: Последовательное выполнение пайплайна.  

- **Пакет `commands`**:  
  - **Реализации команд**:  
    - `EchoCommand`, `CatCommand`, `ExitCommand`, `ExternalCommand`, `GrepCommand`, `PwdCommand`, `WcCommand`.  
  - **Интерфейс**:  
    - `CommandInterface`:  
      - `execute(args: list[str])` — базовый метод выполнения.  
      - `execute_with_input(args: list[str], stdin: str)` — опциональный метод для пайплайнов.
