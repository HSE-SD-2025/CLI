### 1. Структурная диаграмма

```plaintext
     +-------------------------+
     |       CLI Engine        |
     |-------------------------|
     | - parseInput(String)    |
     | - executeCommands()     |
     | - handlePipelines()     |
     +-------------------------+
                /|\
                 |
                 |
    +----------------------------+
    |     Command Interface      |
    |----------------------------|
    | + execute(String[])        |
    +----------------------------+
                /|\
                 |
        +--------+----------+
        |                   |
+---------------+  +------------------+
|  EchoCommand  |  |   CatCommand     |
+---------------+  +------------------+
| + execute()   |  |  + execute()     |
+---------------+  +------------------+
        |                  |
        |                  |
  +-------------+  +------------------+
  |  WcCommand  |  |  ExternalCommand |
  +-------------+  +------------------+
  | + execute() |  |  + execute()     |
  +-------------+  +------------------+
```

### 2. Описание работы системы

#### Общая структура системы

Система состоит из центрального компонента CLI Engine, который управляет разборами строк, выполнением команд и передачей данных через пайплайны. CLI Engine отвечает за:

- Парсинг входной строки в список команд и аргументов.
- Идентификацию и управление переменными окружения.
- Обработку пайплайнов между командами, что позволяет командам взаимодействовать через потоки ввода-вывода.

#### Команды и пайплайны

- Представление команд и пайплайнов: Каждая команда должна реализовать интерфейс Command, в котором определяется метод execute(String[]). Команды могут быть внутренними (например, EchoCommand, CatCommand, WcCommand) или внешними, которые запускаются через процесс.

- Создание команд: При парсинге входной строки CLI Engine создает объекты команд на основе распознанных ключевых слов, например, при встрече "echo" создается объект EchoCommand.

- Выполнение команд: Команды выполняются последовательно или в виде пайплайнов, где выходные данные одной команды передаются в качестве входных другой.

- Потоки в пайплайне: В CLI Engine реализуется логика, управляющая потоками между командами в пайплайне. Это обеспечивается соединением стандартных потоков вывода и ввода команд, через которые данные передаются при помощи пайпов.

#### Разбор входной строки и подстановки

- Разбор строки: CLI Engine содержит метод parseInput(String), который анализирует входной текст, разделяет его на команды и аргументы, учитывая одинарные и двойные кавычки для управления строками и подстановками переменных окружения.

- Подстановки: На этапе парсинга система распознает переменные окружения (начинающиеся с $) и выполняет их подстановку соответствующими значениями.

#### Переменные окружения

- Представление переменных окружения: Переменные сохраняются в Map<String, String>, где ключ — это имя переменной, а значение — ее текущее значение. CLI Engine обрабатывает команды назначения (например, имя=значение) и обновляет соответствующее хранилище переменных.

#### Многопоточность

Поскольку каждая команда в пайплайне может быть выполнена параллельно, потребуется управление потоками ввода-вывода. Это важно для достижения корректного соединения данных между командами в пайплайне, чтобы не было блокировок.
